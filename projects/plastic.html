Autonomous Plastic Sorting

Developing a computer vision pipeline and state machine to control a UR5 robotic arm for real-time plastic pollution cleanup.

View Source on GitHub

Overview: Introduction to Robotics

Developed as the capstone for MIT's Introduction to Robotics (2.12), this project required integrating a Universal Robots UR5 arm with a RealSense LiDAR camera to autonomously identify, track, and sort plastic bottles moving along a conveyor belt.

Custom Hardware & Gripper Prototyping

To interface with the system, I engineered a custom ESP32-S3 hardware controller featuring a 4-button layout and joystick. Our team initially debated between an underactuated gripper and a vacuum gripper. Because the vacuum system allowed for faster fabrication, I developed a wireless testing rig to quickly validate the pneumatic design.

I wrote custom firmware utilizing the ESP-NOW protocol, allowing the joystick controller to transmit low-latency actuation commands to the pump and flush the pneumatic lines while maintaining strict hardware safety interlocks.

Autonomy and State Machine Logic

While my teammate developed the computer vision pipeline to calculate the real-world velocity and center of mass of the bottles, I architected the UR5 control logic using a robust state machine.

When a bottle is detected, the arm breaks idle, predicts the intersection point on the moving conveyor, and drives downward. Upon contact, the vacuum actuates and the arm pulls up. Crucially, the system utilizes force feedback: if the arm crashes into the table, or if it fails to register the expected payload weight upon retraction, it automatically aborts and returns to idle. If the grab is verified, it sorts the payload into the designated bin.

Kinematic Control & Real-World Edge Cases

To execute the state machine, we explicitly opted to write custom velocity control loops rather than relying on UR's native position control, as velocity control proved significantly faster and more flexible for dynamic, moving targets.

Real-world deployment often exposes edge cases. During the final competition, severe and unpredictable stage lighting variations disrupted our computer vision pipeline, limiting our successful grabs. Despite this sensor degradation, our robust hardware and control architecture secured a 3rd place finish, and I was honored to be voted Most Valuable Engineer by my team.
